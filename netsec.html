{~}Introduction to Network Security{~}

Moving into a new residence with network and hardware vulnerabilities exacerbated the need to learn more about IT and NetSec. With no prior knowledge of internet networking, all the protocols seemed daughting. UDP, ICMP, TCP, SSH, IPv4/IPV6, IPSec, DNS, Proxy, Reverse Proxies, where does it end? Does it even end? Do not fret, let's simplify all the craziness. The table below gives a brief overview of the OSI Model or the Open Systems Interconnection Model. These layers are where all computer communication takes place.








LayerNameExamples/Protocol
1 Application HTTP, FTP, SMTP
2 Presentation JPEG, GIF, MPEG
3 Session Appletalk, Winsock, SSH
4 Transport TCP, UDP
5 Network IP, ICMP
6 Data Link Ethernet,ATM
7 Physical Physical components

Understanding the flow of internet traffic is great for troubleshooting problems in the network. In order for computers to commiuncate with each other they need to have a unique identifier, a way for each device to be distinguished separately. These unique identifiers are assigned by a router provided by your ISP. All network traffic needs to be filtered, generally, through one location. The router catalogs every device in promixity with an IPv4 or IPv6 IP address. The figure below gives a brief overview of the IP protocol.



ProtocolDetailsExample
IPv44 octets w/ 32-bit numbers192.168.1.1
IPv66 octets w/ 128-bit alphanumeric2001:4fd0:3850:38g4:876d:38fd

After each device is assigned with an IP address, it allows the cross communication with devices and the internet. The traffic is filtered through the router and off to the ISP to goes. Everytime a request is made through the internet (Think of visiting a website, streaming music, searching for an image, sending an email, etc) a connection must take place initially. Whenever an image or a link on a website is clicked a request is made. There are many different protocols, ICMP, TCP, UDP, ARP, to name a few. The table below gives a brief description of the most known internet protocols.






AbreviationNameDescription
ICMPInternet Control Message Protocol communication between apps.
TCP Transmission Control Protocol errors and statistics (connection confirmation).
UDP User Datagram Protocol communication between apps.
IPInternet Protocolcommunication between devices.
ARPAddress Resolution Protocolused by IP to find hardware.

XSS attacks, clickjacking, SQL injection, etc. Nexxus is also a great tool to harden a web domain. The utility offers warnings and recommendations in order to harden the web domain under scrutiny. There There are many different computer languages that go into crafting a webtsite. There is also a lot of cross communication between these languages on a client and server side.   • HTML (Hyper Text Markup Language): base frame of website.
  • CSS (Cascading Style Sheet): styling for website.

{~}ClamAV{~}

You're probably not as paranoid but I found this utility rather useful. This can be installed with "apt-get install clamav" but requires a couple of steps to get ClamAV to run properly.



Another amazing feat of ClamAV is it does not have to scan the entire computer system, with the ability to scan directories and individual files, ClamAV gives a range of security, from a system wide scan to a sketchy file downloaded from the interwebs.


Below are some of the hiccups I had during install. ClamAV is a rather simple install "sudo apt install Clamav". Unfortunately, there are a couple steps to get this service up and running. I have found that after the apt install a serivce named "freshclam" begins to run. This is a script to check to see if ClamAV is up to date and a quick configuratioin with the system hardware. After install if the command "ps aux | grep freshclam" is executed you will notice it running. The only way I could get freshclam to work is completely restart the freshclam service. "sudo pkill -15 -x freshclam" was used to stop the freshclam scan and then "sudo freshclam" allowed for the update of ClamAV. Another hiccup I came across was clamscan needs access" I had to change the UID to 125 and the GUIDD to 127

Helpful ClamAV commands

After creating a directory for ClamAV (sudo mkdir ~/Desktop/ClamScan) you have the abililty to scan and send all of the suspect files to a folder. Linux everything is set up as it's own file. This is extremely beneficial for various reasons. This makes it much more difficult for the spread of viruses. Linux allows you can contain the suspect files in an almost quaratined folder. With the proper persmissions set for this directory it isolates the files away from the system. I setup a Crontab in order for ClamAV to run at 3am once a week.


The crontab above will run clamscan with -r which will remove the files. The next piece "--move=" will move the suspect files. "~/Desktop/ClamScan" will be the location to send the files and "/*" specifies ClamAV to scan the entire root directory.
